Developing Job Assignment Algorithms

Methods

runJobAssignment(eesa.core.JobSet js, eesa.core. Processor[] processors)
This algorithm performs the assignment.
You are passed the JobSet, on which getJobs() can be called to retrieve all the Jobs. The second argument provides you will an array list of all the processors. The return value is a set of ProcessorResult[] objects each of which should be constructed as follows, to ensure they are attached to a processor: 
ProcessorResult procResult = new ProcessorResult(processor)
If your algorithm has specific conditions regarding the jobs or encounters an error, it is acceptable to throw a eesa.exceptions.AlgorithmException (which should be constructed with a java.lang.String message detailing the problem for the user. This will be presented accordingly by the UI.
Return Type: eesa.core.ProcessorResult[]
toString()
Although a default version of this is created by Java, it is a good idea to override this. It is used as the name of the algorithm in multiple locations, such as when selecting algorithms.
Return Type: java.lang.String
getDescription()
This returns a long description of the algorithm. Things commonly included here are the paper in which it was first described, the authors and remarks on running time and optimality.
Return Type: java.lang.String
