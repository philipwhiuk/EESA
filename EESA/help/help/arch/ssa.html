Developing Speed Scaling Algorithms

Methods

runSpeedScaling(eesa.core.Test t, eesa.core.ProcessorResult p)
This algorithm performs the speed scaling.
The first argument is the Test for which you are running the speed scaling. can be called to retrieve all the Jobs. The second argument provides with the ProcessorResult. To run the algorithm, retrieve the jobs for the Processor (p.getUnassignedJobs()), create eesa.core.JobResult and eesa.core.ProcessorSpeedTimePeriod objects and add them to the ProcessorResult object. If your algorithm has specific conditions regarding the jobs or encounters an error, it is acceptable to throw a eesa.exceptions.AlgorithmException (which should be constructed with a java.lang.String message detailing the problem for the user. This will be presented accordingly by the UI.
Return Type: void
toString()
Although a default version of this is created by Java, it is a good idea to override this. It is used as the name of the algorithm in multiple locations, such as when selecting algorithms.
Return Type: java.lang.String
getDescription()
This returns a long description of the algorithm. Things commonly included here are the paper in which it was first described, the authors and remarks on running time and optimality.
Return Type: java.lang.String
