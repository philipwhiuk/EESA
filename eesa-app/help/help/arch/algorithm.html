Algorithm Loading

The simulator allows additional algorithms to be loaded. To develop an algorithm a number of steps must be followed.

Assuming you are using an IDE to develop the algorithm:

  1. Create a Java Project
  2. Add 'EESA-Core' as a project library
  3. Create a new class for each algorithm, implementing the desired Algorithm interface (eesa.core.JobAssignmentAlgorithm or eesa.core.SpeedScalingAlgorithm).
  4. Compile the project into a JAR file
  5. Place the .jar file in the directly in which EESA is installed
  6. Append the .jar file to the list in the EESA.bat file
  7. Append the name of the algorithm(s) to the related .dat file in the properties folder

The algorithm will then be loaded and can be used as normal.

Note: You may include other libraries (but remember to add them to .bat file and folder). However do not rely on other EESA modules (such as the GUI) as future updates will not attempt to maintain backwards compatability for the purposes of algorithm support.

If you make an error in one of the steps, the simulator will inform you (to the best of it's ability) either via a warning window, or in the log files.
